Tipo de perfil

Debido a que se permite exportar en distintos estándares de e-learning, existen distintos perfiles de evaluación y adaptación. La elección del perfil está relacionada con el modelo de datos existente en el LMS utilizado, haciendo las pertinentes restricciones sobre el mismo. A día de hoy, podemos encontrar tres tipos:

  1. Perfil normal

    En este tipo de perfil no encuentra limitación ninguna sobre el modelo de datos del LMS y se utiliza para aquellos estándares que no poseen modelo de datos específico (como Learning Design).

  2. Perfil SCORM 1.2

    Limita el modelo de datos para el estándar SCORM 1.2 (más información en: http://www.adlnet.gov/). Cuando se decidan modificar o consultar propiedades del LMS con motivo de que se haya cumplido una regla de evaluación o adaptación, solo nos permitirá elegir entre las partes del modelo de datos SCORM 1.2 que son de escritura o lectura, según el perfil. De esta manera se impide almacenar o consultar valores en partes del modelo de datos que son sólo de lectura o sólo de escritura respectivamente, a la vez que abstrae el conocimiento exacto de dicho modelo de datos. Si deseamos exportar como SCORM 1.2, y tenemos perfiles de evaluación y/o adaptación asignados a cualquiera de los capítulos, será requerido que dichos perfiles sean de este tipo.

  3. Perfil SCORM 2004

    Actúa de manera idéntica que para el estándar SCORM 1.2, pero para el modelo de datos del estándar SCORM 2004 (más información en: http://www.adlnet.gov/) . Si deseamos exportar como SCORM 2004 o como AGREGA (los juegos exportados a AGREGA utilizan SCORM 2004), y tenemos perfiles de evaluación y/o adaptación asignados a cualquiera de los capítulos, será requerido que dichos perfiles sean de este tipo.
